# PMBOK第7版実践活用術

## 6-06: チェンジマネジメントのフレームワーク

組織のチェンジマネジメント

- 変革の方向づけ: なぜ必要なのか、将来的にどのように改善するか
- 変革の計画
- 変革の実施: 未来の状態に必要な能力を示し、それが意図した影響を持つことを確認し、必要に応じて改善や適応を行う
- 移行のマネジメント: 変革を達成したあとのニーズに対処する方法を検討する
- 変革の持続: 新たな能力を適用し、古いプロセスや行動を廃止することを継続して行っていく

個人が変化に適応する際にたどる5つの連続したステップ

1. Awareness: 変化が必要な理由を特定する
2. Desire: 変化を支持し、変化の一翼を担う意欲を持つ
3. Knowledge: 変化の方法、新しい役割、責任、プロセスを理解する。
4. Ability: 実践的な訓練や専門知識を通じて知識をサポートする
5. Reinforcement: 変化の持続をサポートするための報奨やフィードバックを行う。

逆にトップダウンによる組織変革の手順:

1. 危機意識の高揚
2. 推進チームの結成
3. 変革ビジョンの作成
4. ビジョンの周知
5. 障害の除去
6. 短期的成果の実現
7. 成果の活用
8. 企業文化への変革定着

バージニア・サティアの変化モデル: 人々の変化の捉え方

1. 初期の現状
2. 異質の要素
3. 混沌
4. 変容のアイデア
5. 実践と統合
6. 新しい現状

## 7-01: プロジェクトに品質を組み込む

品質: 「要求事項への適合」and/or「仕様に適したもの」
PMは品質管理の方針を定め、問題発見と修正のための時間を計画に組み込む。

プロジェクトではテストや検査の結果を記録し、プロジェクトが品質を満たしながら順調に進行していることの証拠とする。問題が発生した場合はその修正と効果の監視を行う。

QA (Quality Assurance) 管理できる人が要求事項の作成に関与することが重要。

## 7-02: プロダクトとプロジェクトの品質管理

**プロダクト** マネジメントの品質管理とは、「プロジェクトの **成果物** が顧客の要求や仕様に合致しているかを確認するための活動」である。

- 成果物の品質とテスト
- 問題の特定と是正処置
- データ分析と報告

**プロジェクト** マネジメントの品質管理とは「プロジェクトの **プロセス** を適切に設計し、品質基準に沿って小売的に進行していることを確実にするための活動」である。

- プロセスの監査とレビュー
- 品質管理計画の策定と実施
- 継続的改善

## 7-03: 品質向上の取り組みと品質管理によるプロセス改善

PMにおける品質向上の取り組みでは以下の点を明確にする。

### 品質基準の設定と遵守

- 品質基準の明確化: PJの成果物が満たすべき品質基準を明確に定義する。
- 基準の遵守

### 継続的な品質基準の評価

- 定期的な品質レビュー
- 品質問題の早期発見と対応

### 品質向上のためのプロセス改善

- プロセスの最適化
- フィードバックの活用

### チームメンバーの能力向上

- 継続的な教育と訓練
- コミュニケーションの強化

### ステークホルダーとの連携

- ステークホルダーとのコミュニケーション
- ステークホルダーの参加を促進

PJにおける品質管理 (成果物がステークホルダーの供給と一致していることの保証)

### プロセス改善へのアプローチ

- 品質計画の策定
- データ収集と分析
- 是正処置と予防処置の実施

チームとのコミュニケーション

### 持続的な改善

- 改善文化の促進
- レビューと反省

## 7-04: 品質に組織全体でコミットする

プロジェクトの品質は技術的側面だけではなく、組織のコミットメントに密接に関連する。組織文化や価値観、業務プロセス、リーダーシップの影響を受けるためである。

### 組織文化の強化

### 継続的な品質改善

- プロセスの見直しと改善

### 全従業員の関与

- 品質意識の高揚

### ステークホルダーとのコミュニケーション

- 透明性と協力
